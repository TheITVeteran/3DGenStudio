# Auto Retopo

Rebuild your mesh with clean, evenly-distributed topology. Auto Retopo is ideal for turning a noisy, high-poly or badly-triangulated generation into a lighter, well-behaved mesh that is easier to texture, edit, and animate.

> Auto Retopo runs on the **Python mesh-tools service**. Make sure it is configured and running in **Settings → Mesh Tools** before using this mode.

## Running it

1. Open the **Auto Retopo** mode in the left toolbar.
2. Set your target face budget and tune the options below.
3. Click **Run Auto Retopo** and wait for the service to finish.
4. The result replaces the current mesh and reports metrics. Click **Keep** to accept it, or **Revert** to restore the previous mesh.

## Parameters

### Target

| Field | Default | What it does |
| --- | --- | --- |
| **Target faces** | 6000 | Approximate face budget of the output mesh. |
| **Quad-dominant** | Off | Convert to quad-dominant topology. Reported in the metrics, but the exported GLB stays triangulated. |

### Watertight shell

When enabled, the mesh is rebuilt from a unified voxel shell instead of remeshing the surface directly. This is the robust option — it heals cracks, self-intersections, and non-manifold geometry — at the cost of some fine detail and speed.

| Field | Default | What it does |
| --- | --- | --- |
| **Watertight shell** | On | Build a unified voxel shell (robust) vs. remesh the surface directly. |
| **Shell resolution** | 256 | Voxel grid cells along the longest bounding-box axis. Higher = more detail, more memory. |
| **Close iterations** | 1 | Morphological closing steps to bridge cracks and small gaps. |
| **Smooth (sigma)** | 1.4 | SDF blur sigma in voxels — kills voxel ripple. Lower = crisper. |
| **Taubin polish** | 10 | Taubin smoothing steps on the dense shell (0 disables). |
| **Samples / pitch** | 2 | Voxel sampling density. ≥ 2 guarantees gap-free coverage. |
| **Max memory (GB)** | 4 | Auto-lower the shell resolution to fit this budget (0 disables the cap). |

*All shell fields are only active when **Watertight shell** is on.*

### Remesh

Controls how the surface is re-tessellated.

| Field | Default | What it does |
| --- | --- | --- |
| **Curvature-adaptive** | On | Place more faces where the surface bends and fewer on flat areas. |
| **Remesh iterations** | 10 | Number of remeshing passes. |
| **Feature angle** | 30° | Crease angle preserved as a feature during remeshing. |
| **Calibrate passes** | 1 | Rough edge-length correction passes. |

### Feature preservation

Hard-surface mode for mechanical / man-made objects with sharp edges.

| Field | Default | What it does |
| --- | --- | --- |
| **Preserve features** | Off | Keep sharp creases crisp and skip smoothing/projection. |
| **Feature angle** | 25° | Crease angle treated as a hard edge. *(Only when preserve features is on.)* |

### Silhouette projection

After remeshing, the new mesh can be projected back onto the original surface to recover its silhouette.

| Field | Default | What it does |
| --- | --- | --- |
| **Project to surface** | On | Project the remesh back onto the original surface. |
| **Projection iterations** | 10 | Number of projection passes. |
| **Move clamp** | 1.5 | Maximum per-vertex move, as a multiple of local edge length. |
| **Relax strength** | 0.4 | Tangential relaxation factor applied each iteration. |

**Note on Preserve features + Projection:**
- In **surface mode** (watertight off), projection is **skipped** when Preserve features is on.
- **Watertight shells are always projected** onto the original surface, even with Preserve features on.

### Misc

| Field | Default | What it does |
| --- | --- | --- |
| **Seed** | 0 | RNG seed for reproducible results. |

---

The result replaces the mesh; use **Keep** or **Revert** to decide.
