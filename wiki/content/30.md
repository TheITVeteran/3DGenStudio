# Auto Retopo

Rebuild your mesh with clean, evenly-distributed topology. Auto Retopo is ideal for turning a noisy, high-poly or badly-triangulated generation into a lighter, well-behaved mesh that is easier to texture, edit, and animate.

> Auto Retopo runs on the **Python mesh-tools service**. Make sure it is configured and running in **Settings → Mesh Tools** before using this mode.

## Running it

1. Open the **Auto Retopo** mode in the left toolbar.
2. Set your target face budget and tune the options below.
3. Click **Run Auto Retopo** and wait for the service to finish.
4. The result replaces the current mesh and reports metrics. Click **Keep** to accept it, or **Revert** to restore the previous mesh.

## Check if Watertight

Click **Check if Watertight** to analyze the current mesh topology. It reports whether the mesh is closed, and if not, how many **open edges** (used by a single face) and **non-manifold edges** (shared by more than two faces) it has.

This check is worth running first: if the mesh is already watertight you can turn the **Watertight shell** rebuild *off* and stay much closer to the original detail (see below).

## Clean Non-Manifold Edges

When the check finds a small number of **non-manifold edges**, you often don't need a full rebuild. A **Clean Non-Manifold Edges** button appears that repairs the topology directly on the Python service: it welds coincident vertices, drops duplicate/degenerate faces, then resolves the non-manifold edges and (optionally) closes the small holes that opens. It runs the same Keep/Revert round-trip as Auto Retopo and reports the before → after non-manifold and open-edge counts.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Repair method** | Remove faces | *Remove faces* deletes the offending faces — best when the non-manifold bits are junk (doubled or floating faces). *Split vertices* detaches the overlapping sheets instead, keeping every face but leaving clean open edges — best when the faces are real geometry you don't want to lose. |
| **Close resulting holes** | On | Seal the small holes that face removal opens. **Leave on** to aim for a closed result. **Turn it off** to guarantee no *new* non-manifold edges are introduced (at the cost of leaving the holes open) — useful when closing a hole would itself re-create a non-manifold junction. |
| **Max hole size** | 30 | Largest hole (in boundary edges) to close. **Lower it** to patch only tiny cracks and leave real openings (a mouth, an armhole) intact. **Raise it** to seal bigger gaps — but too high will close legitimate boundaries. |

> Some meshes have genuinely non-manifold structures (for example, four surface patches meeting at one edge, or doubled "floating" faces) that cannot be resolved into a perfectly closed mesh automatically. In those cases the tool still reduces the defect count and reports honestly what it achieved — use the **Watertight shell** rebuild below if you need a guaranteed closed result.

## Parameters

### Target

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Target faces** | 6000 | Approximate face budget of the output mesh. **Higher** keeps more detail but produces a heavier mesh; **lower** gives a lighter mesh that drops fine features. Set it by purpose — a low budget for game/real-time LODs, a high one for a hero asset you'll sculpt or texture closely. |
| **Quad-dominant** | Off | **Turn it on** when you care about edge flow — quad-dominant topology subdivides and deforms better in a DCC tool. Note the result is only *reported* as quads in the metrics; the exported GLB is still triangulated. Leave it **off** if you only need clean triangles. |

### Watertight shell

When enabled, the mesh is rebuilt from a unified voxel shell instead of remeshing the surface directly.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Watertight shell** | On | **On** is the robust option — it rebuilds from a voxel shell, healing cracks, self-intersections, and non-manifold geometry and guaranteeing a closed mesh, but it loses some fine detail and is slower. **Turn it off** to remesh the existing surface directly and stay far closer to the original detail — safe only when the mesh is already clean/watertight (check first). |
| **Shell resolution** | 256 | Voxel grid cells along the longest bounding-box axis. **Higher** captures finer detail but memory and time grow roughly with the cube of this number — big jumps get expensive fast. **Lower** gives a coarser, blobbier shell quickly. Raise it for detailed models, within your memory budget. |
| **Close iterations** | 1 | Morphological closing steps that bridge cracks and small gaps. **Raise it** only if the shell still has holes — too high fills in concavities and thin gaps you may want to keep (e.g. the gap between fingers). **0** applies no bridging. |
| **Smooth (sigma)** | 1.4 | SDF blur sigma in voxels — kills the voxel "staircase" ripple. **Lower it** for crisper, sharper output (better for hard-surface); **raise it** for smoother output (better for organic blobs), at the cost of rounding off sharp features. |
| **Taubin polish** | 10 | Taubin smoothing steps on the dense shell — smooths the surface without shrinking it. **Raise it** if the shell looks bumpy; **0** disables it. |
| **Samples / pitch** | 2 | Voxel sampling density. **≥ 2** guarantees gap-free coverage. **Raise it** to safely capture very thin features (slower); going below 2 risks leaks or holes on thin parts. |
| **Max memory (GB)** | 4 | Auto-lowers the shell resolution to fit this budget. Set it to the RAM/VRAM headroom you have — **higher** allows a higher effective resolution. **0** disables the cap (the shell may then run out of memory). |

*All shell fields are only active when **Watertight shell** is on.*

### Remesh

Controls how the surface is re-tessellated.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Curvature-adaptive** | On | **On** places more faces where the surface bends and fewer on flat areas — spends your face budget where detail matters (recommended). **Turn it off** for uniform density everywhere, e.g. when you want evenly-sized faces regardless of shape. |
| **Remesh iterations** | 10 | Number of remeshing passes. **Higher** gives more even edge lengths and better-quality topology, but is slower; **lower** is faster and rougher. |
| **Feature angle** | 30° | Crease angle preserved as a feature during remeshing. **Lower it** to treat gentler bends as features worth keeping (preserves more edges); **raise it** so only very sharp creases survive and everything else is smoothed. |
| **Calibrate passes** | 1 | Rough edge-length correction passes that help the face count land near your target. **Raise it** if the output count strays far from *Target faces* (slightly slower). |

### Feature preservation

Hard-surface mode for mechanical / man-made objects with sharp edges.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Preserve features** | Off | **Turn it on** for hard-surface / man-made objects with crisp edges (bolts, panels, blades) — it keeps corners from being rounded off and skips the smoothing/projection that would soften them. **Leave it off** for organic, smooth shapes, where forcing hard edges would create ugly faceting. See the note below on how it interacts with projection. |
| **Feature angle** | 25° | Crease angle treated as a hard edge. **Lower it** to keep more edges sharp; **raise it** so only the sharpest corners are preserved. *(Only when preserve features is on.)* |

### Silhouette projection

After remeshing, the new mesh can be projected back onto the original surface to recover its silhouette.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Project to surface** | On | **Keep it on** — this snaps the rebuilt mesh back onto the original surface, recovering the detail and silhouette lost to remeshing or voxelization. **Turn it off** to leave the remesh/shell as-is (blobbier, but faster and more uniform). |
| **Projection iterations** | 10 | Number of projection passes. **Higher** hugs the original surface more tightly (slower); **lower** gives a looser fit. |
| **Move clamp** | 1.5 | Maximum per-vertex move, as a multiple of local edge length. **Higher** lets vertices travel far to recover deep detail — but can spike if the projection misfires. **Lower it** for a conservative, stable fit; lower it further if you see spikes or shards in the result. |
| **Relax strength** | 0.4 | Tangential relaxation applied each iteration. **Higher** evens out the triangle distribution along the surface (smoother mesh, but can blur fine detail); **lower** keeps vertices where they projected, which may let them cluster. |

**Note on Preserve features + Projection:**
- In **surface mode** (watertight off), projection is **skipped** when Preserve features is on — and the projection fields are disabled.
- **Watertight shells are always projected** onto the original surface, even with Preserve features on.

### Compute

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Device** | Auto (GPU if NVIDIA) | Where the heavy stages run. **Auto** uses an NVIDIA GPU when one is available and falls back to CPU otherwise. Choose **CUDA** to force the GPU, or **CPU** to force CPU (use this if the GPU runs out of memory, or you have no NVIDIA card). Only the watertight-shell stage (CuPy) and the surface-projection stage (NVIDIA Warp) are GPU-accelerated — the remesh / "building clean topology" stage always runs on the CPU regardless of this setting. |

### Misc

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Seed** | 0 | RNG seed for reproducible results. The same seed reproduces the same output; change it only to roll a different random variation. |

---

The result replaces the mesh; use **Keep** or **Revert** to decide.
