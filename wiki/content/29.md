# Auto UV

Automatically unwrap your mesh into a clean UV atlas — no manual seam painting required. Auto UV segments the surface into charts, parameterizes each one, and packs them into a single texture atlas.

> Auto UV runs on the **Python mesh-tools service**. Make sure it is configured and running in **Settings → Mesh Tools** before using this mode.

## Running it

1. Open the **Auto UV** mode in the left toolbar.
2. Adjust the parameters below (the defaults work well for most organic meshes).
3. Click **Run Auto UV** and wait for the service to finish.
4. The result replaces the current mesh and shows a preview of the UV layout. Click **Keep** to accept it, or **Revert** to go back to the previous mesh.

The result panel also reports metrics (chart count, distortion, etc.) so you can compare before committing.

## The core trade-off

Almost every parameter trades between two competing goals:

- **Fewer seams** — cleaner-looking texturing, but the flattened charts *stretch* (distortion), so painted detail looks squashed in places.
- **Less distortion** — every part flattens faithfully, but the surface is cut into *more charts*, so there are more seams to hide.

Reach for the segmentation and refinement settings below when you need to push one way or the other.

## Parameters

### Segmentation

Controls how the surface is cut into charts before unwrapping.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Normal-cone cap** | 50° | Maximum angular spread of face normals allowed within one chart. **Raise it** for fewer, larger charts with fewer seams — good for smooth organic surfaces, but expect more stretch. **Lower it** for more, smaller charts that flatten with very little distortion, at the cost of more seams. Lower this first if the texture looks stretched. |
| **Sharp-edge weight** | 0.35 | How strongly sharp edges attract seams. **Raise it** to snap seams onto hard creases so they hide where the eye already expects an edge — ideal for hard-surface / mechanical models. **Lower it** to place seams purely by distortion, ignoring creases — better for organic shapes with no real edges. |
| **Fold cap** | 88° | Dihedral fold angle that *forces* a seam: any edge bending more than this is always cut. **Lower it** to add more forced cuts, which stops charts from folding over themselves on crumpled or self-touching geometry. **Raise it** for fewer forced cuts on clean meshes. Lower it if charts overlap in the preview. |
| **Min faces / chart** | 20 | Charts with fewer faces than this are dissolved into their neighbours. **Raise it** to sweep away tiny sliver charts for a tidier atlas (a small chart merged into a big one may pick up some distortion). **Lower it** to preserve small detail charts as their own islands. |
| **Min area fraction** | 0.004 | Same idea as *Min faces*, but measured by area — the minimum chart area as a fraction of the total surface. **Raise it** to aggressively cull small charts; **lower it** to keep them. Use it to kill slivers that survive the face-count test. |

### Refinement

An optional pass that merges small charts back together when merging doesn't add too much distortion.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Validated merge pass** | On | Runs an LSCM-validated chart merge. **On** gives fewer charts and a tidier atlas but is slower. **Turn it off** for a faster unwrap, or when you deliberately want many small, low-distortion charts. |
| **Merge below faces** | 80 | Charts smaller than this face count are candidates for merging. **Raise it** to merge more aggressively (fewer charts, but a bit more distortion). **Lower it** so only the very smallest charts are absorbed. *(Only when the merge pass is on.)* |
| **Merge distortion cap** | 1.32 | The most angle-distortion a merge is allowed to introduce — a merge is only accepted if it stays under this ratio. **Raise it** to tolerate more stretch in exchange for fewer seams. **Lower it** to keep distortion tight and leave more charts. Lower it if merged areas look stretched. *(Only when the merge pass is on.)* |

### Parameterization

How each chart is flattened into 2D.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Method** | Auto | Flattening algorithm. **Auto** picks a suitable method per chart (recommended). **LSCM** is angle-preserving and fast, a solid default, but can stretch *areas*. **ARAP** (as-rigid-as-possible) balances angle and area for the least visible stretch, but is slower. **Planar** is a flat projection — only use it for near-flat charts (panels, decals). Switch to ARAP if LSCM shows area stretch. |
| **ARAP iterations** | 4 | Number of As-Rigid-As-Possible relaxation passes. **Raise it** to reduce distortion further (with diminishing returns and more time). **Lower it** for speed. **0** disables ARAP entirely (LSCM / planar only). |

### Packing

How the charts are arranged into the final atlas.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Atlas resolution** | 1024 px | Target atlas size (256 → 8192 px). **Higher** captures more texture detail with less pixelation, at the cost of larger files and more memory. **Lower** is lighter but softer. Match it to how close the model will be seen and your texture budget. |
| **Padding** | 4 texels | Gutter between charts. **Raise it** to stop colours bleeding across chart edges when the texture is mip-mapped or viewed at a distance — but padding eats usable atlas space. **Lower it** to pack tighter and reclaim resolution, at the risk of seam bleed. Note the gutter scales with resolution, so higher-res atlases often want a few more texels. |

### Topology repair

Cleans up broken geometry before unwrapping.

| Field | Default | What it does & how to tune it |
| --- | --- | --- |
| **Proximity weld** | On | Welds coincident vertices before unwrapping. **Keep it on** — AI-generated meshes are often shattered along invisible cracks, and welding stitches those shells back together so charts don't fragment. **Turn it off** only when the mesh is already clean and you need to preserve exact vertex splits. |
| **Weld tolerance** | 0.1 | Weld distance, as a fraction of the median edge length. **Raise it** to fuse vertices that sit further apart (closes bigger gaps, but can collapse genuine fine detail). **Lower it** to fuse only near-exact duplicates. Raise it if shells still fragment; lower it if the weld is merging things it shouldn't. *(Only when weld is on.)* |

---

The result replaces the mesh; use **Keep** or **Revert** to decide.
